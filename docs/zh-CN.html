Plugin · MCP · Skill 检查器

看清你的会话到底加载了什么。

一个零依赖的 Claude Code 插件，读取你磁盘上的配置，为每一个 plugin、MCP、skill、agent、hook 打印字节 / token 账本 —— 并告诉你哪一个参数决定了 300 个 skill 的开销是 2k 还是 20k tokens / turn。

01 · 工作原理

在下一个 turn 开始前，先做一次磁盘静态审计。

/context 看的是当前会话。/pms-inspector 读你 ~/.claude/ 里的配置，预测下一个会话会加载什么 —— 不调 LLM、不写盘、不联网。

  1. 1

    发现

    遍历 enabledPlugins 与磁盘上的每一个 MCP、skill、agent、hook。

  2. 2

    度量

    对 frontmatter + description 计字节，按 4 字符 ≈ 1 token 换算。

  3. 3

    分类

    套用 skillListingBudgetFraction 与 skillListingMaxDescChars。四种状态，界限清晰。

  4. 4

    报告

    人眼对齐的表格（含 CJK 视觉宽度）或 --json 供流水线使用。

  5. 5

    建议

    给出具体的参数值 + token 差额 —— 复制粘贴进 settings.json。

02 · 四种加载状态

每一个 skill 都必然落进一格。

这四张色卡采用 Pantone 风格 —— 因为 PMS 恰好也是 Pantone Matching System。这个工具做的事，本质上就是把你发布的和真正被加载的做一次校色。

PMS 5773 CFULL

描述完整送达模型 —— 你希望被触发的 skill 就该在这一格。

PMS 138 CDESC-TRUNCATED

被 skillListingMaxDescChars（默认 1536）截断。末尾被静默切掉。

PMS 165 CBUDGET-COMPRESSED

被全局 skillListingBudgetFraction 挤压。文本还在，但比源文件短。

PMS 179 CNAME-ONLY

描述被完全丢弃 —— 模型只看到 slug，无从判断何时该调用它。

03 · 参考案例 · 318 skills · 200k 窗口

两组参数。差距大约 17k tokens。

之前 —— 默认 budget 0.01

318 / 318 name-only

只花了约 2 000 tokens，但模型不知道任何 skill 存在的理由。一半调用打不中。

之后 —— budget 调到 0.15

318 / 318 full

约 18 900 tokens（窗口的 9.4%）。每一个 skill 的触发描述都送达模型。调用开始命中。

before · fraction 0.01
skills     318 discovered
  name-only      318   ← 100 %
  full             0

model sees:  318 slugs, zero context
est. cost:   ~ 2.0 k tokens/turn
after · fraction 0.15
skills     318 discovered
  full           318   ← 100 %
  name-only        0

model sees:  318 slugs + full descriptions
est. cost:  ~ 18.9 k tokens/turn  (9.4 % of window)

04 · 工具真正在看什么

三个旋钮。四种状态。一份真相。

skillListingBudgetFraction
Claude Code 为 skill 描述预留的上下文比例。默认 0.01。skill 一多，这个参数会让所有 skill 一起掉到 name-only。
skillListingMaxDescChars
单条描述在被标记为 desc-truncated 前的硬上限。默认 1536。很少是瓶颈 —— 通常是 fraction 先咬到。
--ctx <tokens>
覆盖假定的上下文窗口（默认 200000）。给 Haiku 或旧 tier 建模时用得上。
--json
机器可解析输出。四种加载状态、字节 / token 计数、每个 skill 明细 —— 直接进你的 pipeline。
--verbose
人类报告里附带每个 skill 的明细。适合排查某一个 skill 占用异常大的情况。
--lang <code>
强制输出语言：en · zh-CN · zh-TW · ja · ko · fr · de · es。不指定则从 settings.json 自动检测。
