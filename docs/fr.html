01 · Fonctionnement

Un audit statique du disque, avant le prochain tour.

/context montre la conversation en cours. /pms-inspector lit votre configuration ~/.claude/ et prédit ce que la prochaine session chargera — pas d'appel LLM, pas d'écriture, pas de réseau.

  1. 1

    Découvrir

    Parcourir enabledPlugins et chaque serveur MCP, skill, agent et hook du disque.

  2. 2

    Mesurer

    Compter les octets du frontmatter + description ; convertir à 4 caractères ≈ 1 token.

  3. 3

    Classer

    Appliquer skillListingBudgetFraction et skillListingMaxDescChars. Quatre états, sans flou.

  4. 4

    Rapporter

    Tableaux alignés pour l'humain (largeurs CJK conscientes) ou --json pour vos pipelines.

  5. 5

    Suggérer

    Valeurs concrètes des réglages avec deltas de tokens — à copier-coller dans settings.json.

02 · Les quatre états de chargement

Chaque skill tombe dans exactement un carré.

Chaque carré est une puce style Pantone parce que PMS désigne aussi le Pantone Matching System — et tout ce que cet outil fait, c'est appairer ce que vous publiez avec ce qui est effectivement chargé.

PMS 5773 CFULL

La description atteint le modèle intacte — l'état idéal pour un skill que vous voulez vraiment voir déclenché.

PMS 138 CDESC-TRUNCATED

Plafonnée par skillListingMaxDescChars (défaut 1536). La fin est coupée en silence.

PMS 165 CBUDGET-COMPRESSED

Compressée par le skillListingBudgetFraction global. Le texte reste, mais plus court que le fichier.

PMS 179 CNAME-ONLY

Description totalement supprimée — le modèle ne voit que le slug et ne peut pas raisonner sur quand l'invoquer.

04 · Ce que l'outil inspecte vraiment

Trois réglages. Quatre états. Une vérité.

skillListingBudgetFraction
Fraction de la fenêtre de contexte que Claude Code réserve aux descriptions de skills. Défaut 0.01. Sur une configuration chargée, c'est le seul réglage qui fait basculer tout le monde en name-only.
skillListingMaxDescChars
Plafond dur par description avant qu'elle soit marquée desc-truncated. Défaut 1536. Rarement le goulot — c'est habituellement la fraction qui mord en premier.
--ctx <tokens>
Redéfinir la fenêtre de contexte supposée (défaut 200000). Pratique pour modéliser Haiku ou un tier plus ancien.
--json
Sortie parsable par machine. Les quatre états, comptes octets / tokens, détail par skill — à brancher sur ce que vous utilisez déjà.
--verbose
Détail par skill dans le rapport humain. Utile quand un skill pèse disproportionnellement lourd.
--lang <code>
Forcer la langue de sortie : en · zh-CN · zh-TW · ja · ko · fr · de · es. Sinon détectée depuis settings.json.
