01 · しくみ

次のターンが起動する前に、ディスクを静的に監査する。

/context は現在の会話を見せる。/pms-inspector は ~/.claude/ の設定を読み、次のセッションが何を読み込むかを予測する —— LLM 呼び出しなし、書き込みなし、ネットワークなし。

  1. 1

    検出

    enabledPlugins とディスク上のすべての MCP / skill / agent / hook を走査。

  2. 2

    計測

    frontmatter + description のバイトを数え、4 文字 ≒ 1 トークンで換算。

  3. 3

    分類

    skillListingBudgetFraction と skillListingMaxDescChars を適用。4 状態、あいまいなし。

  4. 4

    レポート

    人間向けの整列テーブル (CJK 幅対応)、またはパイプライン向けの --json。

  5. 5

    提案

    具体的なつまみの値とトークン差分 —— そのまま settings.json に貼れる。

02 · 4つの読み込み状態

すべての skill は必ず 1 マスに落ちる。

各マスは Pantone 風のチップ —— PMS は Pantone Matching System でもあり、このツールがやっているのは、あなたが公開したものと実際に読み込まれたもののカラーマッチングだから。

PMS 5773 CFULL

description がモデルに完全に届く —— トリガーさせたい skill が本来居るべき状態。

PMS 138 CDESC-TRUNCATED

skillListingMaxDescChars (既定 1536) で頭打ち。末尾が黙って切られる。

PMS 165 CBUDGET-COMPRESSED

グローバルな skillListingBudgetFraction に圧縮される。テキストは残るが、原文より短い。

PMS 179 CNAME-ONLY

description が完全に落ちる —— モデルには slug しか見えず、いつ呼ぶべきかを判断できない。

04 · このツールが実際に見るもの

つまみ 3 つ。状態 4 つ。真実 1 つ。

skillListingBudgetFraction
Claude Code が skill の description に確保するコンテキスト割合。既定は 0.01。skill が多いと、これ 1 つで全 skill が name-only に落ちる。
skillListingMaxDescChars
desc-truncated と印される前の 1 description あたりの上限。既定 1536。ボトルネックになることは少ない —— 通常は fraction のほうが先に噛む。
--ctx <tokens>
想定するコンテキストウィンドウを上書き (既定 200000)。Haiku や古い層をモデリングするときに便利。
--json
機械可読出力。4 つの読み込み状態、バイト / トークン数、skill ごとの内訳 —— そのままパイプラインに投げられる。
--verbose
人間向けレポートに skill ごとの内訳を追加。特定の skill が突出して重いときに便利。
--lang <code>
出力言語を強制: en · zh-CN · zh-TW · ja · ko · fr · de · es。指定しなければ settings.json から自動検出。
