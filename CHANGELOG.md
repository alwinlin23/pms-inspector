# Changelog

All notable changes to `pms-inspector` are recorded here. Format follows
[Keep a Changelog 1.1.0](https://keepachangelog.com/en/1.1.0/) and versioning
follows [SemVer](https://semver.org/spec/v2.0.0.html).

The plugin itself (`scripts/inspect.js`, slash-command wiring, i18n dictionary)
is versioned via git tags. Documentation and site-only changes accumulate under
`[Unreleased]` and roll into the next tagged release.

## [Unreleased]

### Added
- `docs/_template.html`, `docs/_i18n.json`, `docs/_build.mjs` — single-source
  i18n build. All 8 localized pages (`index.html` + 7 language variants) are
  now regenerated by `node docs/_build.mjs`; do not hand-edit the outputs.
- Zero-dep constraint extended to the site build. `_build.mjs` uses Node
  builtins only (`fs`, `path`, `url`) — same shipping rule as `inspect.js`.
- `📄 Website:` link (localized in 7 languages: `网站 / 網站 / ウェブサイト /
  웹사이트 / Site web / Website / Sitio web`) at the top of each translated
  README, pointing at the Pages site.
- `CLAUDE.md` gets an **Editing the docs site (GitHub Pages)** section that
  documents the `_i18n.json` → `_build.mjs` → 8 files workflow and the
  don't-hand-edit rule.

### Changed
- Rewrote `docs/style.css` around the Anthropic warm-neutral palette
  extracted from `claude.com/product/claude-code` (clay `#d97757` accent,
  gray-050/100/300/500/800/900 stops). Fonts: **Fraunces** (display),
  **Inter** (body), **JetBrains Mono** (terminal) via Google Fonts CDN as
  free stand-ins for Anthropic's proprietary `anthropicSerif`/`Sans`/`Mono`.
- Editorial rebuild of the Pages layout: retired `.states / .state / .stepper
  / .costs / .flow / .ba / .cost-card`; introduced `.band / .hero / .term /
  .method / .swatches / .pms-swatch / .two-cost / .cta-band`.
- The four skill-load states now render as **Pantone-style swatches**
  (`FULL`, `DESC-TRUNCATED`, `BUDGET-COMPRESSED`, `NAME-ONLY`) — the P/M/S
  brand double-reads as *Pantone Matching System*, matching what shipped vs.
  what actually loaded.
- Language switcher moved to a **flag + native-name dropdown** rendered by
  the build script (`{{langDropdown}}` placeholder).

## [0.2.2] — 2026-07-11

### Fixed
- Report body now streams through the assistant message (single render)
  instead of the Bash tool output box plus a re-render below it. Slash-command
  markdown change only; `inspect.js` unchanged.

## [0.2.1] — 2026-07-11

### Fixed
- `commands/pms-inspector.md` fallback resolver for `$CLAUDE_PLUGIN_ROOT` —
  Claude Code sets the var for frontmatter substitution but does not always
  propagate it into the Bash tool subprocess. Fallback walks the CC-mandated
  cache path so `node "$PMS_ROOT/scripts/inspect.js"` never collapses to
  `/scripts/inspect.js`.

## [0.2.0] — 2026-07-10

### Added
- i18n across 8 languages (`en / zh-CN / zh-TW / ja / ko / fr / de / es`) with
  auto-detection order: `--lang` flag → `~/.claude/settings.json` `language` →
  `$LC_ALL / $LANG / $LANGUAGE` env → `en` fallback.
- Accepts CC-native language names (e.g. `"简体中文"`, `"日本語"`,
  `"français"`) via `CC_LANGUAGE_ALIASES`.
- East-Asian-Width aware column arithmetic (`scripts/lib/width.js`) —
  `padEndVisual` / `padStartVisual` / `truncateVisual` for correct table
  alignment on any mix of ASCII / CJK / emoji.
- `--verbose` per-skill breakdown.
- `--ctx <tokens>` override for the assumed context window.

## [0.1.0] — Initial release

### Added
- `/pms-inspector` slash command — static audit of enabled plugins, MCP
  servers, skills, agents, and hooks under `~/.claude/`. Byte / token math
  and concrete tuning suggestions for `skillListingBudgetFraction` /
  `skillListingMaxDescChars`. Zero npm deps.
- Four-way skill classifier (`full / desc-truncated / budget-compressed /
  name-only`) — the entire pipeline routes through this state.
- `--json` machine-parseable output.
